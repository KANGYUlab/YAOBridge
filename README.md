# YAOBridge: repeat- and conservation-aware framework for high-confidence variant projection between hg38 and T2T-YAO

## 1. Motivation

Accurate cross-genome mapping at base resolution is essential for translating telomere-to-telomere (T2T) assemblies into functional insight, particularly for projecting curated annotations such as ClinVar from legacy references (e.g., hg38) to population-specific T2T genomes. Existing approaches, including Liftover and transAnno, rely on block-based mappings that lack base-level fidelity and degrade in repeat-rich regions, limiting their applicability to structurally complete genomes. Here we present YAOBridge, a repeat- and conservation-aware framework that integrates alignment filtering with genome stratification based on gene boundaries, synteny, and sequence identity. Built on a PAF-based chaining architecture, YAOBridge resolves multi-copy ambiguity and enforces base-level consistency across genomes, enabling accurate projection of variants and functional elements, including within complex and repetitive loci. YAOBridge establishes a robust foundation for variant interpretation in T2T coordinate systems and facilitates the systematic transfer of genomic knowledge across references.

## 2. Core Algorithm

### 1) Genome stratification

We define **T2T-YAO (maternal haplotype + chrY)** as the query genome and stratify its sequence into four hierarchical region classes based on cross-genome conservation with hg38 and additional high-quality T2T assemblies:

**·** **Level 1 (conserved framework)**: blocks of single-copy orthologous genes and flanking sequences that preserve synteny and exhibit high sequence identity and coverage across diverse T2T human genomes.

**·** **Level 2 (conserved inter-block regions)**: regions between Level 1 blocks that maintain synteny and high identity/coverage between YAO and hg38, with unique alignments.

**·** **Level 3 (ambiguous regions)**: inter-block regions with reduced identity/coverage or multiple valid alignments between YAO and hg38.

**·** **Level 4 (unmapped regions)**:regions that cannot be reliably aligned between YAO and hg38.

### 2) Base-level projection

We construct **base-resolution projections** between YAO and hg38 using PAF alignments generated by minimap2. The projection is **repeat-aware**, retaining all candidate mappings in multi-aligned regions rather than enforcing a single best hit.

### 3) Projection scoring

Each projected position is assigned a **Confidence Score (CS)** reflecting conservation, alignment quality, mappability, and synteny, calculated as the sum of four components:

**S1: Conservation Level**

Reflects the hierarchical classification of the genomic region:

- **Level 1**: Score = **4    **   **Level 2**: Score = **3**       **Level 3**: Score = **2**           **Level 4**: Score = **1**

**S2: Alignment Quality**

Evaluates the local sequence fidelity:
$$
S2 = 2 \times \text{identity} \times \text{coverage}
$$
*(Calculated based on inter-block alignment metrics)*

**S3: Mappability**

Penalizes multi-mapping or repetitive regions:
$$
S3 = \frac{2}{N_{\text{alignments}}}
$$
**S4: Synteny Consistency**

Assesses the structural stability of the mapping:

- **2**: Position maps to the **same inter-block region**.
- **1**: Position maps to the **same chromosome** but different block.
- **0**: Position maps to a **different chromosome** or is unmappable.

***Final projection score:***
$$
S = S1 + S2 + S3 + S4
$$

# 4.Usage

### **Installation**

```bash
git clone https://github.com/KANGYUlab/PreciseBridge.git
cd pcctool
pip install .
```



### Options

```bash
$ YAOBridge -h
usage: YAOBridge [-h] -s {hg38,yao} -t {hg38,yao} -file FILE -out OUT [-V]

YAOBridge: hg38 <-> yao mapping

options:
  -h, --help     show this help message and exit
  -s {hg38,yao}  source Genome
  -t {hg38,yao}  target Genome
  -file FILE     #chrom #pos
  -out OUT       output file path
  -V, --version  show program version
```

### Output

Output Data Format

The output is a tab-delimited file where each row represents the alignment result for a specific genomic locus, organized into four main segments:

a. Locus Coordinates: Query chromosome name and base position.

b. Status Flag: Alignment status indicator (Found or NotFound).

c. 1-to-1 Mapping: High-confidence primary alignment result (None if unavailable).

d. 1-to-N Mapping: Semicolon-delimited list of all potential multi-copy alignment results (None if unavailable).
